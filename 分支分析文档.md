# Feiyizhan.github.io 仓库分支分析文档

## 分支概览

该仓库包含三个主要分支，每个分支都有不同的用途：

### 1. master 分支（主分支）

**作用：** 源代码分支，包含最新的博客文章、配置文件和模板文件。

**特点：**
- 包含完整的 Jekyll 源代码
- 包含所有博客文章（`_posts/` 目录）
- 包含网站配置（`_config.yml`）
- 包含布局模板（`_layouts/`）
- 包含数据文件（`_data/`）
- 包含静态资源（`assets/`）

**用途：**
- 作为主要的开发分支
- 用于本地开发和测试
- 包含最新的文章和功能更新

**最新提交：**
- `test: 测试置顶文章修改`
- `fix:修复tops文章的URL不正确的BUG`
- `feat: 增加新的置顶文章`

### 2. gh-pages 分支（GitHub Pages 分支）

**作用：** 用于 GitHub Pages 网站托管，包含已编译的静态网站文件。

**特点：**
- 包含已生成的静态 HTML 文件
- 通常由 Jekyll 自动生成
- 用于 GitHub Pages 服务托管网站
- 访问地址：`https://feiyizhan.github.io/`

**用途：**
- GitHub Pages 自动从该分支读取文件并托管网站
- 当 master 分支更新时，GitHub Pages 会自动构建并更新该分支

**注意：**
- 该分支的提交历史较旧，最后一次更新是关于主题样式的修改
- 如果使用 GitHub Actions 或 Travis CI，会自动将构建结果推送到此分支

### 3. gitee-pages 分支（Gitee Pages 分支）

**作用：** 用于 Gitee Pages 网站托管，包含针对 Gitee 优化的配置。

**特点：**
- 包含源代码和配置
- 集成了 Giteement 评论系统（基于码云的评论系统）
- 配置了 Gitee OAuth 认证
- 访问地址：`https://feiyz.gitee.io/`

**用途：**
- 用于在 Gitee Pages 上托管网站
- 使用 Giteement 评论系统，适合国内用户访问
- 包含针对 Gitee 的特殊配置

**特殊配置：**
- `giteement.enable: true` - 启用码云评论系统
- `giteement.giteeID: 'FeiYZ'` - 码云账号
- `giteement.repo: 'FeiYZ'` - 评论存储仓库
- 配置了 OAuth 客户端 ID 和 Secret

**最新提交：**
- `feat: 切换gitment为giteement`
- `feat: 增加gitee主页`

## 分支关系图

```
master (源代码)
  ├── gh-pages (GitHub Pages 托管)
  └── gitee-pages (Gitee Pages 托管 + Giteement)
```

## 工作流程建议

1. **开发流程：**
   - 在 `master` 分支进行开发和内容更新
   - 添加新文章、修改配置等操作都在 `master` 分支完成

2. **部署流程：**
   - **GitHub Pages：** 通常由 GitHub Actions 自动构建并推送到 `gh-pages` 分支
   - **Gitee Pages：** 手动或通过 CI/CD 将 `master` 分支内容同步到 `gitee-pages` 分支

3. **内容管理：**
   - 所有内容修改应在 `master` 分支进行
   - 使用本地的 `pages` 文件夹进行本地开发和测试
   - 使用 `manage_content.py` 脚本进行内容管理

## 注意事项

1. **不要直接编辑 `gh-pages` 分支：** 该分支通常由构建工具自动生成
2. **`gitee-pages` 分支的特殊性：** 包含 Giteement 配置，需要保持与 `master` 的同步
3. **本地开发：** 建议在 `master` 分支或本地 `pages` 文件夹中进行开发
